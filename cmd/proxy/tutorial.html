🔐 oauth4os Interactive Tutorial

Learn OAuth 2.0 by doing — 5 steps, real API calls, curl equivalents

1
Register a Client

Every OAuth application starts by registering a client. You'll get a client_id and client_secret. Or use the registration form.

curl equivalent
curl -X POST /oauth/register \ -H "Content-Type: application/json" \ -d '{"name":"my-tutorial-app","scopes":["read:logs-*"]}'
2
Get an Access Token

Exchange your client credentials for an access token. This is the client_credentials grant type.

curl equivalent
curl -X POST /oauth/token \ -d "grant_type=client_credentials&client_id=YOUR_CLIENT_ID&client_secret=YOUR_SECRET"
3
Make an Authenticated Request

Use your token to search logs. The proxy validates the JWT, checks scopes, and forwards to OpenSearch.

curl equivalent
curl -H "Authorization: Bearer YOUR_TOKEN" \ /logs-demo/_search?q=*&size=3
4
Revoke the Token

Revoke the token immediately. After this, the same token will return 401 Unauthorized.

curl equivalent
curl -X POST /oauth/revoke \ -d "token=YOUR_TOKEN"
5
Try the PKCE Flow

PKCE (Proof Key for Code Exchange) is the recommended flow for public clients (SPAs, CLIs). It prevents authorization code interception.

How PKCE works
1. Generate code_verifier (random 43-char string) 2. Compute code_challenge = BASE64URL(SHA256(verifier)) 3. Redirect to /oauth/authorize?code_challenge=... 4. User approves → redirect back with ?code=AUTH_CODE 5. Exchange: POST /oauth/token with code + code_verifier 6. Server verifies SHA256(verifier) == challenge
6
Refresh the Token

Use a refresh token to get a new access token without re-authenticating. The old refresh token is rotated.

curl equivalent
curl -X POST /oauth/token \ -d "grant_type=refresh_token&refresh_token=YOUR_REFRESH_TOKEN&client_id=YOUR_CLIENT_ID"
