Architecture

oauth4os sits between clients and OpenSearch — validating tokens, mapping scopes, enforcing policies.

Quickstart — 30 Seconds

1. Start

docker run -p 8443:8443 jianghuan/oauth4os:latest

2. Get a scoped token

curl -X POST http://localhost:8443/oauth/token \
  -d "grant_type=client_credentials" \
  -d "client_id=my-agent" \
  -d "client_secret=secret" \
  -d "scope=read:logs-*"

3. Query OpenSearch

curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8443/logs-*/_search \
  -d '{"query":{"match":{"level":"error"}}}'

4. Or use the CLI

oauth4os login --provider keycloak
oauth4os create-token --scope "read:logs-*"
oauth4os status

Features

    

      
🔑
JWT Validation
JWKS auto-discovery, RS256/ES256, audience validation, clock skew tolerance.

      
🎯
Scope → Role Mapping
read:logs-* → OpenSearch logs_reader. Per-provider mappings for multi-tenancy.

      
🌲
Cedar Policies
Fine-grained access control. Index-level deny rules. Per-tenant policy sets.

      
⚡
Rate Limiting
Per-client token bucket. Configurable RPM per scope. 429 + Retry-After.

      
📋
Token Lifecycle
Issue, refresh, revoke, introspect (RFC 7662). Refresh token rotation with reuse detection.

      
🔄
PKCE Flow
Secure browser auth (RFC 7636). Authorization code + code_verifier exchange.

      
👥
Multi-Tenancy
Per-OIDC-provider isolation. Separate scope mappings and Cedar policies per tenant.

      
📊
OSD Plugin
Token management UI in OpenSearch Dashboards. List, create, revoke with EUI components.

      
🤖
MCP Server
Reference AI agent integration. 7 tools: search, aggregate, create index, mappings.

      
⌨️
CLI Tool
login, create-token, revoke, status. Config file, token caching, auto-refresh.

      
📈
Prometheus Metrics
/metrics endpoint. Pre-built alerting rules. Grafana dashboard included.

      
🔍
Structured Logging
JSON audit logs. X-Request-ID tracing. Every proxied request logged.
